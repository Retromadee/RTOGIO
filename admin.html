Notifications Guide
How to notify customers
WhatsApp — Click the WhatsApp button next to any order. It opens a pre-filled WhatsApp message addressed to the customer. Just send it.

Email — Click the Email button to send an automatic status-update email via EmailJS. Requires EmailJS to be configured in js/config.js.

Advance Status — Click Advance → to move the order to the next stage. The customer's tracking page updates instantly via Firebase. Combine with WhatsApp/Email to notify them.
Security & "Leaked" Keys
Why your Firebase keys in config.js are safe to be public
It is completely normal and safe for Firebase web API keys to be visible in Chrome DevTools. They simply identify your Firebase project. To actually secure your data, you MUST apply rules in your Firebase Console (Realtime Database -> Rules tab):

{
  "rules": {
    "inventory": {
      ".read": true,
      ".write": false
    },
    "orders": {
      // Anyone can create an order or read if they know the exact ID
      "$orderId": {
        ".read": true,
        ".write": "!data.exists() || data.child('proofOfPayment').exists()"
      }
    }
  }
}
📝 Note: These are basic rules that solve the immediate "public key" worry. For a perfect production app, you should eventually implement Firebase Authentication to truly lock down the admin access points.
